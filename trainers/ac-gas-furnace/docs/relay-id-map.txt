======================================================
STATE CONVENTION
======================================================
state=1 => energize command (firmware writes LOW for relay outputs)
state=0 => de-energize command (firmware writes HIGH for relay outputs)

NO/NC CONTACT BEHAVIOR (FOR EACH PHYSICAL RELAY)
- state=1 (coil energized): COM->NO closes, COM->NC opens
- state=0 (coil de-energized): COM->NO opens, COM->NC closes
- If a field circuit is currently wired on NC, expected field behavior is inverted relative to NO wiring.
- Verify board silkscreen and terminal labeling for each installed relay module before commissioning.

Notes:
- This file is furnace-specific (trainers/ac-gas-furnace).
- New relay_* entries above are aliases that reuse existing f* hardware lines.
- Efficiency pattern: use relay_* aliases for instructor controls and keep f* IDs for diagnostic/fault simulation workflows.
- Shared line caveat: relay_rollout_limit_2 and relay_failed_gas_relay currently target the same physical line (f34 / board_1 pin 11) on this hardware profile.
- Independent-channel note: relay_grounded_w_wire, relay_shorted_y_to_r, relay_shorted_w_to_r, and relay_shorted_contactor_coil now use independent simulation channels (f52/f51/f53/f54).
